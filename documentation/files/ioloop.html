ioloop.IOLoop:new ()
Create a new instance of IOLoop. IOLoop is a class responsible for managing I/O events through file descriptors with epoll. Heavily influenced by ioloop.py in the Tornado web server. Add file descriptors with :add_handler(fd, listen_to_this, handler). Handler will be called when event is triggered. Handlers can also be removed from the I/O Loop with :remove_handler(fd). This will also remove the event from epoll. You can change the event listened for with :update_handler(fd, listen_to_this). Warning: Only one instance of IOLoop can ever run at the same time!

Usage:

local ioloop = ioloop.IOLoop:new()

Return value:

IOLoop class instance.
